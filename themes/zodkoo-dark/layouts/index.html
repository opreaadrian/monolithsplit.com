There is a better way!

Stop adding features to that old monolith and start creating microservices.

That's right! Although it sounds like nonsense, stopping the feature-bloat on your current application is the way to make more money. Start splitting that huge chunk of code into money-making services!

Remember that mobile application you wanted to build? Did the work to create separate APIs for your data prove too big of a challenge? With microservices you can think of a mobile app, a web app or a desktop app as a simple representation to your data! You could even create a command line interface for the terminal-lover in your customer.

Technology's fast-paced landscape makes it harder and harder to launch features in a 6 months release cycle. With certain companies deploying multiple times a day, it's hard to keep up if you're too far behind the pack.

Microservices enable you to deploy as many times you like and perform experiments that would otherwise seem impossible due to the big "blast radius" a monolithic architecture involves.
